# Аналитика продаж — Интерактивный дашборд

Современный адаптивный дашборд для анализа продаж на React + TypeScript.  
Данные загружаются из Excel-файла. Архитектура позволяет подключить CRM без изменения компонентов.

## Технологии

- **React 19** + **TypeScript**
- **Vite** — сборка
- **Tailwind CSS** — стилизация
- **Recharts** — графики
- **Zustand** — управление состоянием
- **SheetJS (xlsx)** — импорт/экспорт Excel

## Запуск

```bash
npm install
npm run dev
```

Открой `http://localhost:3000`.

## Сборка для production

```bash
npm run build
npm run preview
```

## Источник данных

При старте дашборд автоматически загружает `Sample_Sales_Data_MVP_Extended.xlsx` из корня проекта.  
Можно также загрузить файл вручную через интерфейс (drag & drop или кнопка «Выбрать файл»).

При изменении Excel-файла в папке перезагрузи страницу — данные обновятся.

## Структура страницы

1. **Загрузчик файла** — выбор Excel или drag & drop
2. **Фильтры** — период, регион, категория, текстовый поиск
3. **KPI-карточки** — 5 ключевых метрик
4. **Динамика продаж** — линейный график (продажи + прибыль)
5. **Продажи по регионам** — столбчатая диаграмма (все регионы, игнорирует фильтр региона)
6. **Распределение по категориям** — кольцевая диаграмма с долями в % (все категории, игнорирует фильтр категории)
7. **Автоматические выводы** — инсайты по данным
8. **Экспорт** — выгрузка отфильтрованных данных в Excel

## Архитектура

```
src/
├── api/apiService.ts      # Адаптер для будущего API/CRM
├── components/             # UI-компоненты
├── store/useStore.ts       # Zustand-хранилище
├── types/index.ts          # TypeScript-типы
└── utils/
    ├── calculations.ts     # Вычисления KPI, фильтры
    └── dataParser.ts       # Парсер Excel с валидацией
```

Для перехода с Excel на CRM достаточно реализовать методы интерфейса `DataSource` в `apiService.ts` — компоненты не требуют изменений.
